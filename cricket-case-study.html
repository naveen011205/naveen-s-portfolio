CricScore – Cricket Scoring App

Role: UI/UX Designer

Duration: 2 Weeks

Tools: Figma, FigJam

Platform: Mobile & Web

Problem Statement

Cricket fans, players, and local tournament organizers often rely on multiple apps or manual scorekeeping to follow matches. Many existing scoring apps are cluttered with complex interfaces, making it difficult to quickly update scores or view important match information.

The goal was to design a clean, fast, and intuitive cricket scoring application that allows users to score matches effortlessly while giving spectators easy access to live scores, player statistics, and match insights.

Project Goal

Design a user-friendly cricket scoring application that enables users to:

  • Create and manage cricket matches
  • Record ball-by-ball scores quickly
  • View live score updates
  • Track player and team statistics
  • Access match history
  • Deliver a responsive experience across mobile and web

Target Users

Primary Users

  • Local cricket scorers
  • Tournament organizers
  • Team captains

Secondary Users

  • Cricket fans
  • Coaches
  • Players
  • Friends and family following live matches

User Research

Research included reviewing existing cricket scoring applications and identifying common usability issues.

Key Findings

  • Navigation was often confusing
  • Important information was hidden behind multiple screens
  • Ball-by-ball scoring required too many taps
  • Live score updates were difficult to read
  • Statistics pages were overcrowded
  • New users found onboarding difficult

User Persona

Design Decisions

Large Scoring Buttons

Enabled scorers to update scores quickly with fewer errors during live matches.

Simplified Navigation

Reduced the number of taps needed to access important features.

High Contrast Layout

Improved readability in bright outdoor conditions.

Consistent Components

Designed reusable buttons, cards, and input fields to maintain visual consistency.

Challenges

  • Organizing large amounts of match data without overwhelming users
  • Designing a fast scoring interface suitable for real-time use
  • Balancing detailed statistics with a clean and simple layout

Outcome

The final design delivers a modern and intuitive cricket scoring experience by simplifying live score entry, improving information hierarchy, and making match statistics easier to understand. The interface is responsive, visually clean, and optimized for both scorers and spectators.

Key Learnings

  • Simplicity improves usability during time-sensitive tasks
  • Information hierarchy is essential for sports applications
  • Consistent design systems speed up UI development
  • User-centered design reduces friction and increases efficiency
  • Rapid prototyping helps identify usability issues early in the design process
